Multiple Inheritance dengan Mixin

  • Multiple inheritance: digunakan agar satu objek mewarisi perilaku/properti dari dua atau lebih class sekaligus.
  • Dart: pakai mixin untuk menggabungkan fungsionalitas beberapa class tanpa pewarisan rumit.
  • Bahasa yang mendukung: C++, Python, Scala, Perl (multiple inheritance langsung), Java, C#, Kotlin (single inheritance).
  • Contoh: Bebek (berenang + terbang), Smartphone (kamera + telepon).

Studi Kasus: Burung dan Bebek

Mixin digunakan untuk menggabungkan beberapa kemampuan (tanpa inheritance yang rumit).

// Nama  : Putri Ananda Amelia
// NIM   : 251410029
// Kelas : SI2A

mixin CanFly {
  void fly() {
    print('Can fly!');
  }
}

mixin CanSwim {
  void swim() {
    print('Can swim!');
  }
}

class Bird with CanFly {}

class Duck extends Bird with CanSwim {}

void main() {
  Bird bird = Bird();
  bird.fly();

  Duck duck = Duck();
  duck.fly();
  duck.swim();
}

Studi Kasus: mixin dengan state

Mixin juga bisa punya state (variabel) untuk menyimpan kondisi.

// Nama  : Putri Ananda Amelia
// NIM   : 251410029
// Kelas : SI2A

mixin CanFly {
  bool _isFlying = false;

  bool get isFlying => _isFlying;

  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print('Started flying!');
    } else {
      print('Already flying!');
    }
  }

  void land() {
    if (_isFlying) {
      _isFlying = false;
      print('Landed!');
    } else {
      print('Already on the ground!');
    }
  }
}

mixin CanSwim {
  bool _isSwimming = false;

  bool get isSwimming => _isSwimming;

  void swim() {
    if (!_isSwimming) {
      _isSwimming = true;
      print('Started swimming!');
    } else {
      print('Already swimming!');
    }
  }

  void stopSwim() {
    if (_isSwimming) {
      _isSwimming = false;
      print('Stop swimming!');
    } else {
      print('Already stopped swimming!');
    }
  }
}

class Bat with CanFly {}

class Frog with CanSwim {}

class FlyingFish with CanFly, CanSwim {}

class Dolphin with CanSwim {}

void main() {
  Bat bat = Bat();
  bat.fly();
  bat.land();

  print('Is Bat flying? : ${bat.isFlying}');

  print('----------------');

  Frog frog = Frog();
  frog.swim();
  frog.stopSwim();

  print('Is Frog swimming? : ${frog.isSwimming}');

  print('----------------');

  FlyingFish fish = FlyingFish();
  fish.swim();
  fish.fly();
  fish.land();
  fish.stopSwim();

  print('Is FlyingFish swimming? : ${fish.isSwimming}');

  print('----------------');

  Dolphin dolphin = Dolphin();
  dolphin.swim();
  dolphin.stopSwim();
}

Sealed Class

Sealed class membatasi subclass hanya di file/library yang sama.

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

Final Class

Final class tidak bisa diwarisi dari luar file

final class Settings {
  final String theme;
  final int fontSize;

  Settings({required this.theme, required this.fontSize});

  void showSettings() {
    print("Theme: $theme, Font Size: $fontSize");
  }

base class CustomSettings extends Settings {
  CustomSettings(String theme, int fontSize)

Perbandingan Class Modifiers

  • sealed: Tidak bisa di-extend dan di-implement dari luar file, hanya bisa di dalam file yang sama.
  • base: Bisa di-extend di luar, tetapi tidak bisa di-implement di luar. Di dalam file masih bisa keduanya.
  • final: Tidak bisa di-extend di luar, tetapi masih bisa digunakan di dalam file.
  • (default): Tidak memiliki batasan, bisa di-extend dan di-implement di mana saja.

Praktik 1 : Sealed Class dengan Pattern Matching

Memudahkan pengecekan kondisi dengan switch yang aman.

// Nama  : Putri Ananda Amelia
// NIM   : 251410029
// Kelas : SI2A
  
sealed class LoginStatus {
  const LoginStatus();
}

class LoggingIn extends LoginStatus {
  const LoggingIn();
}

class Success extends LoginStatus {
  final String username;
  const Success(this.username);
}

class Error extends LoginStatus {
  final String message;
  const Error(this.message);
}

void handleStatus(LoginStatus status) {
  switch (status) {
    case LoggingIn():
      print('Logging in...');
    case Success(:final username):
      print('Welcome, $username');
    case Error(:final message):
      print('Login failed: $message');
  }
}

void main() {
  final states = [
    LoggingIn(),
    Success('Putri'),
    Error('Wrong password'),
  ];

  for (var state in states) {
    handleStatus(state);
  }
}

Studi Kasus : Sistem State Management

Digunakan untuk mengatur state aplikasi secara aman.

// Nama  : Putri Ananda Amelia
// NIM   : 251410029
// Kelas : SI2A

sealed class DataState {}

class LoadingData extends DataState {}

class DataLoaded extends DataState {
  final List data;
  DataLoaded(this.data);
}

class DataError extends DataState {
  final String message;
  DataError(this.message);
}

void updateUI(DataState state) {
  switch (state) {
    case LoadingData():
      showLoading();
    case DataLoaded(:final data):
      showData(data);
    case DataError(:final message):
      showError(message);
  }
}

void showLoading() {
  print('⏳ Loading data...');
}

void showData(List data) {
  print('✅ Data loaded:');
  for (int i = 0; i < data.length; i++) {
    print('   ${i + 1}. ${data[i]}');
  }
}

void showError(String message) {
  print('❌ Error: $message');
}

void main() {
  final states = [
    LoadingData(),
    DataLoaded(['Data 1', 'Data 2']),
    DataError('Gagal memuat data'),
  ];

  for (var state in states) {
    updateUI(state);
  }
}

Kesimpulan

Sealed class cocok untuk membuat tipe data dengan hierarki terbatas agar pattern matching aman. Base class digunakan untuk membuat class dasar yang bisa diturunkan di framework atau aplikasi. Final class cocok untuk utility class yang tidak boleh diturunkan atau diimplementasi dari luar.

// math_utils.dart
final class MathUtils {
  static int tambah(int a, int b) {
    return a + b;
  }

  static int kurang(int a, int b) {
    return a - b;
  }

  static int kali(int a, int b) {
    return a * b;
  }

  static double bagi(int a, int b) {
    return a / b;
  }
}

  • Menjalankan method dari MathUtils
  • Tidak perlu membuat objek
